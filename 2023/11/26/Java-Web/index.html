Java Web程序

JSP基础：

JSP的处理过程：
客户端浏览器发出请求——服务器加载被请求的JSP文件——Web服务去将JSP文件转化为Java代码——JSP引擎将生成的Java代码编译成Class类——服务期执行该Class文件——发送到客户端浏览器显示
JSP的优点：
简便性和有效性、程序的独立性、程序的兼容性、程序的可重用性。

（一）、JSP指令标识

1、page指令：用于设置当前页面配置信息。(<%page %>)
2、include指令，用于将其他页面包含到当前页面中。(<%include %>)
3、taglib指令：用于导入标签库。

（二）、JSP脚本标识：

JSP中的脚本标识包括三部分：
即声明标识、JSP表达式和ScriptLet脚本程序。
1、声明：<%! %>
2、JSP表达式：<%=%>
3、ScriptLet脚本：<% Java代码 %>
4、注释标识：①、JSP标准注释<%– –%>没有源码，没有流量
` ②、利用Java语言本身的注释 <%! //这是注释 %>
③、HTML注释()页面看不到，源代码中可以看到，会增加流量数据

JSP内置对象：

（一）、JSP内置对象介绍：

分类如下：
1、输入和输出对象：用于完成请求参数的接收处理和返回响应，包含有request、response、out三个对象。
2、作用域通信对象：用于在web项目中存储和访问数据，包含有（作用域从小到大）：pageContext、session、application三个对象。
3、Servlet对象：用于配置和读取jsp页面的一些参数，包含有：page、config两个对象。
4、错误对象：用于在jsp页面中获取一些错误信息，包含有exception一个对象。
JSP表单处理——request对象：
用来包含客户端提交的各种信息。主要的一个作用就是用来封装用户提交的表单信息，然后来获取用户提交的表单信息。
常用的获取参数方法有：
（1）、String getParameter(String name)：根据参数名获取唯一的参数值，返回字符串类型的结果。
（2）、 String[] getParameterValues(String name)：根据参数名获取唯一的参数数组，常用于CheckBox复选框，全选然后删除的情况。
post提交方式乱码：request.setCharcterEncoding(“utf-8”);
5、作用域通信对象：
常用的作用域通信对象有三个：
1、存储范围最小的pageContext对象，存入它里面的数据，只能在同一个jsp页面才能取出使用。
2、session对象，可以跨越很多页面存储和取出数据，但多个页面资源必须处于同一回话中。
3、application对象，存储的数据作用域最大，整个服务器范围都有较，只要Tomcat服务器不关闭，将一直存在，且所有的客服端共享该数据。（存储大，时间久，一般不会用）
作用域通信对象的两个常用方法：①存储数据方法：setAttribute(“名字”,”数据”);
②取出数据方法：getAttribute(“名字”);取出时数据类型会变 为Object需要自己转一下。
6、exception对象：
exception对象用来处理JSP文件执行时发生的所有错误和异常，只有zaipage指令中设置isErrorPage属性值为true的页面中才可以被使用，在一般的JSP页面中该对象将无法编译JSP文件。
getMessage()：返回异常的信息。
getLocalizedMessage()：返回本地化的异常错误
toString()返回关于异常错误的简单信息描述
fillInStackTrace()：重写

JavaBean和JSP标准动作：

JavaBean是使用Java编写的组件，没一个JavaBean实现一个特定功能。通过合理地组织不同功能的JavaBean，可快速地生成一个全新的应用程序。
    好处：提高代码可重用性、可维护性、易维护性。
    要求：有一个public无参数构造方法、实现java.io.Serializable接口、提供属性接口。

7、按照功能分类：1、封装数据的JavaBean 2、封装业务方法的JavaBean。
8、JavaBean在JSP页面中的使用方法：
（1）、使用Scriptlet脚本代码实现，就是<% 的方式。
（2）、通过JSP标准动作实现。（如：<jsp:userBean id=”对象名” class=”包名” 属性名=”属性值”>）
实例：<jsp:userBean id=”userbean” class=”com.java.util.HashMap”>
使用JavaBean中的set和get方法：
<jsp:setProperty name=”对象名” property=”属性名” value=”固定值”或param=”请求参数名”/>
<jsp:getProperty name=”对象名” property=”属性名”/>
9、界面跳转的两种方法：
（1）、转发（jsp:forward用于转发其他jsp、HTML、servlet等资源）
实例：<jsp:forward page=”url”>
<jsp:include page=”url” flush=”true”>：用于将另一个页面包含到当前Jsp页面实现项目页面中公共的部分的重用。
（2）、重定向

Servlet技术：

1、Servlet技术：
Java Web应用程序的处理过程，实际上是一个不断请求与响应的过程。
流程如下：
（1）、客户端发送请求至Web服务器。
（2）、Web服务器接收请求并进行处理。
（3）、Web服务器将响应结果返回给客户端，并展现用户。
2、Servlet技术：
要使Servlet对象正常地运行，需要进行适当的配置，以通知Tomcat服务器对于哪个请求调用哪一个Servlet对象进行处理，对Servlet起到注册的作用。Servlet的配置信息在web.xml文件中：
（1）、声明Servlet对象
（2）、映射Servlet对象
（3）、运行Web项目

3、Servlet的生命周期：
①加载(第一次请求时) ②实例化(第一次请求时) ③初始化(第一次请求时)
④服务(每次执行) ⑤销毁(服务器停止时)

6、重定向和转发的区别：
JSP中页面自动跳转的方式：
（1）、转发：
1）转发是继续传递request请求，效率高于重定向。
2)转发后地址栏保留旧地址。
3）转发只能在项目内部资源进行。
4）转发通过request对象中的方法进行。
（2）、重定向：
1）重定向是新的请求，所以request中存储的数据，重定向后无法取出。
2）重定向后地址栏改变为新地址。
3）重定向可以跳转到其他服务器及项目外部资源。
4）重定向通过response对象进行。
两者的选择原则：
（1）优先选择转发，因为转发效率更高。
（2）、在同一个Web应用程序的两个请求间传递数据，那只能通过转发实现。
（3）如果需要跳转到其他服务器上的资源，则必须使用重定向。
7、默认配置参数：
1）ServletContext：Servlet上下文坏境对象。它配置的参数所有Servlet都可以取出使用。
2）ServletConfig：提高程序的灵活性和扩展性。
Model1：JSP+JavaBean
Model2：JSP+Servlet+JavaBean
MVC：Model模型、View视图、Contrller控制器
优点：耦合性低、重用性高、部署快、可维护性高、有利于软件工程管理
缺点：不适合小型、中等规模的应用程序、增加系统结构和实现的复杂性、视图与控制器间的过于精密联系。
