Jenkins安装部署

发表于2023-11-26|更新于2023-11-26|Java部署
|阅读量:

安装前提

  • 已安装 jdk
  • 已安装 maven
  • 已安装 nodejs
  • 具有 docker 及 docker-compose 环境

安装

  • 创建 jenkins 目录并进入

mkdir -p /root/docker-compose-data/jenkins/jenkins_home && cd /root/docker-compose-data/jenkins

  • 授权
  • 编写、编辑并保存 docker-compose.yml 文件

vim docker-compose.yml

粘贴进下面内容

  • 执行 docker 命令，创建并启动容器

docker-compose up -d

  • 监听日志，获取管理员密码

docker-compose logs -f

升级

  • 下载新版本 war 包

https://mirrors.tuna.tsinghua.edu.cn/jenkins/war/

  • 将下载的 war 包放在宿主机的 /root 目录下

  • 将 jenkins.war 文件复制到 jenkins 容器的 /usr/share/jenkins 目录下

docker cp /root/jenkins.war containerId:/usr/share/jenkins/jenkins.war # 记得替换容器 id

  • 重启 jenkins

docker restart containerId

配置

  • xxxxxxxxxx docker-compose up -dbash
  • 选择推荐的插件

  • 等待插件下载完成，自动进入创建管理员页面，创建管理员信息并保存

  • jenkins URL 不需要更改，直接保存

配置全局环境

  • 进入系统管理 > 全局工具配置

与镜像挂载一致

/usr/local/maven/apache-maven-3.8.8/conf/settings.xml

/usr/local/jdk11

/usr/local/maven/apache-maven-3.8.8

Publish Over SSH 配置

  • 进入系统管理 > 插件管理

  • 下载并重启 jenkins
  • 进入系统管理 > 系统配置

创建项目及 Dockerfile 文件，上传到 git

  • 注意 Dockerfile 文件在项目根目录

构建

  • 点击添加凭证，输入 git 账户/密码 （如果添加不进去，可以去 系统管理 > 凭据管理 中添加）

  • 如果远程构建，可以将 Dockerfile 文件一起传输过去

插件

- Build Authorization Token Root

触发远程构建 时不需要鉴权：

  • blue Ocean
